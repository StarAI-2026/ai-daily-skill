# 标题生成器规则

## 核心流程

```
输入文章 → 判断文章类型 → 提炼核心观点 → 按两段式结构组装 → 输出 5 条候选标题
```

## Step 1: 判断文章类型

读取文章后，先判断属于以下哪一类：

| 类型 | 特征 | 示例 |
|------|------|------|
| 单篇深度 | 只讲一个项目/技术，深度展开 | 一个 API 调度全球模型 |
| 日报汇总 | 多个项目/事件盘点 | GitHub 今日 3 个开源项目炸了 |
| 测评对比 | A vs B 横向对比 | Cursor vs Claude Code 谁更强 |
| 教程 | 手把手教怎么做 | 5 分钟用 Agent 搭建客服 |
| 种草推荐 | 推荐某个工具/产品 | 这个省 token 的框架真香 |

## Step 2: 提炼核心观点

从文章中找出**最具反差感**或最让目标读者产生"对，就是这样"共鸣的观点。

反差感来源：
- **预期违背**：原本以为 A，结果是 B（如"框架本来是给架构师玩的，现在变成产品了"）
- **效率跃迁**：从 N 变成 N×10（如"同样 token 让 Agent 更聪明"）
- **权力反转**：原本需要专业能力，现在人人可用（如"一行命令搞定长任务"）
- **时代拐点**：行业格局被改变（如"一个 API 调度全球模型，格局变了"）

## Step 3: 两段式结构组装

每条标题必须包含两个部分：

```
[核心观点 / 情绪钩子] + [信息量承诺 / 受众锚定]
```

### 核心观点 / 情绪钩子（前半句）

- 用一句有冲击力的话概括文章最具冲击力的点
- 直白、口语化、像朋友间发的一条消息
- 制造"我想点进来看"的冲动

### 信息量承诺 / 受众锚定（后半句）

- 明确告诉读者"看完能得到什么"
- 用"N 个项目""一文看完""3 分钟搞懂"等数字锚定信息量
- 用"开发者必看""AI 圈动态"等锚定受众

## Step 4: 风格铁律（必须严格遵守）

### ✅ 必须做到

1. **直白**：不用任何修饰词（如"震惊""炸裂""逆天"）
2. **口语化**：像朋友间发的一条消息
3. **不用感叹号堆砌**：最多一个感叹号，或完全不用
4. **每条不超过 25 个字**
5. **聚焦一个核心点**：不贪多，一条标题只讲一件事

### ❌ 禁用清单

| 禁用词/句式 | 原因 |
|------------|------|
| "炸了" / "重磅" / "变天了" | 失效的夸张词 |
| "震惊" / "逆天" / "逆天改命" | 空洞修饰词 |
| "让你大开眼界" / "颠覆认知" | 套话 |
| "必看" / "必读" / "必收藏" | 命令式套话 |
| "99% 的人不知道" | 标题党老梗 |
| "震惊！XXX" | 标题党老梗 |
| emoji | 微信信息流不显示且影响阅读 |

## Step 5: 输出格式

输出 5 条候选标题，并推荐其中最贴合的一条：

```
## 文章类型判断

**[类型名]**：[一句话说明为什么是这个类型]

## 5 条候选标题

1. **[候选1]**（推荐）
2. [候选2]
3. [候选3]
4. [候选4]
5. [候选5]

---

## 推荐说明

选择候选1的原因：[一句话说明为什么推荐这条]
```

## 示例

### 示例 1：日报汇总型

文章内容：3 个 GitHub AI 开源项目盘点（省 token 框架、Rust 极简编程助手、微软 AI 终端）

```
## 文章类型判断

**日报汇总型**：3 个 GitHub AI 开源项目盘点，共同趋势是"从大而全向专而精转变"。

## 5 条候选标题

1. **省一半 token 的 Agent 来了，3 个开源项目一文看完**（推荐）
2. AI 工具不卷功能卷效率了，GitHub 今日爆款一文看完
3. AI 编程从大而全变专而精，3 个开源项目一文看完
4. 这几个 AI 开源项目炸了，GitHub 今日动态一文看完
5. 同样 token 让 Agent 更聪明，3 个开源项目一文看完

## 推荐说明

选择候选1的原因：前半句有具体数字"省一半 token"有冲击力，后半句"3 个开源项目一文看完"锚定信息量。
```

### 示例 2：单篇深度型

文章内容：Sakana Fugu 多智能体编排产品深度分析

```
## 文章类型判断

**单篇深度型**：深度分析一个产品（Sakana Fugu），核心反差是"多智能体从框架变成产品"。

## 5 条候选标题

1. **多智能体终于不是摆设了，一个 API 调度全球模型**（推荐）
2. 框架是积木，产品是钥匙，Fugu 把多智能体变成开箱即用
3. 调用一支智能体团队，一个 API 就够了
4. 多智能体从架构师玩具变成开发者工具
5. 一个 API 搞定多智能体编排，格局变了

## 推荐说明

选择候选1的原因：前半句"终于不是摆设了"有情绪共鸣，后半句"一个 API 调度全球模型"有具体画面感。
```

## 占位符说明

本规则输出的标题将用于 `references/cover-prompt-template.md` 中的 `[主标题]` 占位符。

- 选定 1 条作为**主标题**
- 从文章中提炼 1-3 个关键词作为**副标题**（如"3 个开源项目一文看完"或"省 token=省钱")
