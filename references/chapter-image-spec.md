# 章节配图规范（酸性野兽派 - 图像主导版）

## 与封面的区别

章节配图**保持酸性野兽派视觉语汇**，但**从文字主导转为图像主导**，装饰元素简化，主体是内容相关的图形。

| 维度 | 封面 | 章节配图 |
|------|------|----------|
| 核心视觉 | 文字主导（主标题大字） | 图像主导（数据可视化/流程图/产品概念图） |
| 文字 | 主标题 + 副标题 | 1-2 个关键词点睛（不喧宾夺主） |
| 装饰元素 | 完整语汇（圆圈/闪电/半色调/便签/错版） | 简化为 1-2 种（半色调点 + 错版重影） |
| 配色 | 撞色高饱和 | 与封面同色系但更克制（降低饱和度） |
| 排版 | 字符重叠碰撞 | 主体居中，文字小且克制 |

## 核心规范

| 维度 | 规范 |
|------|------|
| 画面比例 | 强制 16:9 横版 |
| 无人物 | 画面中完全移除人物元素 |
| 核心主体 | 必须是该章节描述的关键事件/产品/数据的图形化呈现 |
| 文字 | 1-2 个关键词点睛，不喧宾夺主 |
| 配色 | 与封面同色系但降低饱和度，保持视觉连贯 |
| 装饰 | 半色调圆点 + 错版印刷重影（2 种即可） |

## 4 种内容图类型

根据章节内容，选择对应类型生成配图：

### 类型 1：事件 / 过程

用**流程图/时间轴/四宫格**展示事件的几个关键步骤。

```
适用：产品发布流程、技术架构演进、安全闭环、Agent 工作流
视觉：分步骤的图形块 + 箭头连接 + 每步一个关键词
```

### 类型 2：人物 / 产品

直接展示**产品的高清概念图/设计图/抽象图形**。

```
适用：新产品发布、模型发布、工具更新
视觉：产品抽象图形（如 API 枢纽、芯片、终端窗口）+ 产品名关键词
```

### 类型 3：数据 / 对比

将抽象数据转化为**柱状图/饼图/折线图/进度条**。

```
适用：性能对比、参数规模、市场份额、增长数据
视觉：简洁数据图表 + 关键数字高亮
```

### 类型 4：场景 / 概念

用**抽象场景插画**概括该章节的核心概念。

```
适用：观点类、趋势类、行业动态类
视觉：抽象场景（如赛车、城市、网络）+ 概念关键词
```

## 占位符替换说明

| 占位符 | 替换为 | 示例 |
|--------|--------|------|
| `[底层色]` | 与封面同配色方案的底层色 | `#2D2D2D 冷灰` |
| `[主色]` | 与封面同配色方案的主色 | `#39FF14 荧光绿` |
| `[辅色]` | 与封面同配色方案的辅色 | `#FF6EC7 脏粉` |
| `[内容类型]` | 4 种类型之一 | `类型1-事件/过程` |
| `[事件名]` / `[步骤N]` | 事件内容 | `Daybreak` / `发现` / `验证` / `修补` |
| `[产品名]` / `[具体图形描述]` | 产品内容 | `Fugu` / `API枢纽节点辐射连接多模型` |
| `[图表类型]` / `[数据内容]` | 数据内容 | `横条图` / `PP-OCRv6性能对比` |
| `[关键数字N]` | 关键数字 | `86.2%` / `83.2%` |
| `[概念]` / `[场景描述]` | 概念内容 | `从大而全变专而精` / `赛车在赛道飞驰` |
| `[关键词]` | 点睛关键词 | `省token` / `闭环` / `极简` |

## 重要约束

1. **章节配图不能与封面使用相同提示词**，否则会选到封面图的变体
2. 每张配图必须包含该主题的**具体内容**，不能只有背景
3. 每个主要章节各生成一张
4. **必须走"下载原图"流程**获取高清原图
5. 配图色系必须与封面**同色系但更克制**（降低饱和度 30%），避免视觉断裂
6. 装饰元素简化为 2 种（半色调点 + 错版重影），避免喧宾夺主

## 风格校验清单

- [ ] 16:9 横版
- [ ] 无人物
- [ ] 核心主体是内容相关图形（非纯背景）
- [ ] 文字只有 1-2 个关键词，字号小
- [ ] 装饰元素只有 2 种（半色调点 + 错版重影）
- [ ] 与封面同色系但更克制
- [ ] 整体是酸性野兽派语汇

如不满足 5 项以上，重新生成。
