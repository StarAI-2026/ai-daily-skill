# Copywriting 文案优化指南

本指南用于在 `article.md` 初稿生成后，对标题、导语、正文进行深度加工，确保文章具备高点击率、可读性和人格化表达。

---

## 一、标题优化

### 1.1 硬约束

- **frontmatter `title` 必须 ≤20 字**（微信公众号信息流完整显示上限）
- **候选标题 ≤25 字**，最终入选 frontmatter 的必须截断至 ≤20 字
- **前 14 字必须包含「主体+动作」**：如"OpenAI 新模型""GitHub 热榜第一"
- 一条标题只讲一个核心点，不贪多

### 1.2 利益导向与具体性

- **利益导向**：不说"发布了 1.0 版本"，而说"开发者的工作流该改改了"
- **具体性**：用数字、场景、结果替代模糊形容词
  - 差：`AI 技术大幅度提升程序员开发效率`
  - 好：`3 款新 AI 工具，把 4 小时报告压缩到 15 分钟`

### 1.3 悬念与结尾钩子

标题末尾 3-5 字制造悬念或行动暗示：

- `，但所有人都忽略了这一点`
- `，你的工作流可能要变了`
- `，背后有一条暗线`
- `，这个趋势藏不住了`

## 二、九维标题评分体系

为每篇文章生成 3-5 个候选标题，按以下 9 个维度打分（每项 1-5 分），选择总分最高者。

| 维度 | 含义 | 高分特征 |
|------|------|----------|
| **A - Attention 注意力** | 0.3 秒内能否抓住眼球 | 有数字、主体、冲突或反常识 |
| **I - Interest 兴趣** | 是否激发阅读兴趣 | 有悬念、场景或读者痛点 |
| **D - Desire 欲望** | 是否创造点击/探究欲望 | 承诺收益、节省时间、获得认知差 |
| **A - Action 行动** | 是否促使立刻点击 | 结尾有钩子，像朋友在分享 |
| **Credibility 可信度** | 是否像人写的，不像营销号 | 有具体事实，无夸张失效词 |
| **Curiosity Gap 好奇缺口** | 是否留下未解答的问题 | 给线索但不给结论 |
| **Emotional Intensity 情绪强度** | 是否有适度情绪但不煽动 | 有态度、有判断，不亢奋 |
| **Specificity  specificity** | 是否具体可感 | 有数字、产品名、场景 |
| **Platform Fit 平台适配** | 是否符合微信公众号语境 | ≤20 字，无 emoji，无话题标签 |

**计分方式**：总分 = A+I+D+A + Credibility + Curiosity + Emotion + Specificity + Platform Fit。满分 45 分。

**最终入选规则**：
1. 先剔除 Platform Fit <4 分或 Credibility <3 分的标题
2. 在剩余标题中选总分最高者
3. 若最高分标题 >20 字，必须截断至 20 字以内，且保留主体+动作

## 三、导语（摘要）优化

### 3.1 硬约束

- **frontmatter `description` 必须 ≤120 字**
- 禁用"综上所述""值得注意的是""让我们"等 AI 味开头
- 禁用 emoji、话题标签、感叹号堆砌

### 3.2 三段式结构

导语必须按 **痛点 → 解法 → 利益** 重写：

1. **痛点（1 句）**：指出读者日常困境，用具体场景
2. **解法（1 句）**：引出今日热点/项目/动态
3. **利益（1 句）**：说明读完能获得什么

**示例**：

> 每天刷 GitHub 看到新项目却不知道怎么选？今天 3 个 AI 开源项目给出了新思路：不卷功能卷效率。看完你会知道哪个工具适合自己，以及怎么最快上手。

## 四、去 AI 味策略

文章写完后必须逐条自检，确保读起来像真人科技 UP 主，而非 AI 生成。

### 4.1 句式层面

| 禁用句式 | 替换方式 |
|---------|----------|
| "值得注意的是" | 直接说事实，或用"这里有个点挺有意思" |
| "不得不说" | "讲真""说实话" |
| "综上所述" / "总而言之" | 直接给结论，或用"说白了" |
| "让我们来看看" | "来看一个""瞅一眼" |
| "首先...其次...最后..." | 用过渡句自然串联 |
| "被开发" / "被发布" | "发布了""推出了" |

### 4.2 词汇层面

| 禁用词 | 原因 | 替换 |
|-------|------|------|
| "重磅" / "史诗级" / "颠覆" / "炸裂" / "炸了" | 营销号失效词 | 用具体数据或事实 |
| "震惊" / "逆天" / "逆天改命" | 空洞夸张 | 描述实际效果 |
| "必看" / "必读" / "必收藏" | 命令式套话 | 不说或改为"值得一看" |
| "99% 的人不知道" | 标题党老梗 | 删除 |
| "赋能" / "闭环" / "抓手" / "颗粒度" | 互联网黑话 | 用大白话 |

### 4.3 节奏层面

- **短句为主**：每句不超过 25 字，超过就拆
- **长短句交替**：重要观点单独成段
- **段落 2-4 句话**：避免大段文字
- **口语化连接词**："那说实话呢""就是说""换句话说""打个比方""讲真""说白了"

### 4.4 视角层面

- 用"我"和"你"，像朋友聊天
- 敢说"我觉得""我认为""说实话我有点失望"
- 不下绝对结论，但可以下明确判断
- 用中国化比喻（小米、华为、比亚迪、大疆、微信、B 站等场景）

## 五、跨平台适配

当前 workflow 主发布渠道为微信公众号，标题和摘要必须按微信规则适配。

| 平台 | 标题限制 | 摘要限制 | 特殊规则 |
|------|---------|---------|----------|
| 微信公众号 | ≤20 字（信息流完整显示） | ≤120 字 | 无 emoji、无话题标签 |
| 知乎（备用） | 建议 ≤40 字 | 建议 ≤150 字 | 可适度使用问号 |
| 今日头条（备用） | 建议 ≤30 字 | 建议 ≤120 字 | 避免过度悬念 |

**当前 workflow 只输出微信公众号版本**，但生成候选标题时可额外输出 1 个知乎/头条适配标题作为备用。

## 六、质量核对清单

保存 `article.md` 前必须自检：

1. [ ] 标题 ≤20 字，前 14 字包含主体+动作
2. [ ] 摘要 ≤120 字，采用痛点→解法→利益结构
3. [ ] 无"值得注意的是""综上所述""让我们"等 AI 味句式
4. [ ] 无"重磅""炸了""颠覆"等失效营销词
5. [ ] 无 emoji、无话题标签
6. [ ] 无长句（每句 ≤25 字）
7. [ ] 被动语态已改为主动语态
8. [ ] 专业术语已替换为中国化比喻或大白话
9. [ ] 图片引用文件名与 `_image_manifest.json` 中一致
10. [ ] 全文朗读一遍，确保不磕巴

## 七、快速参考卡

**标题公式**：

```
[主体 + 动作/变化] + [悬念/收益/数字]
```

**摘要公式**：

```
[读者痛点] → [今日热点/解法] → [阅读收益]
```

**语气校准**：

- 像 B 站科技 UP 主跟朋友聊天
- 有态度、有判断、有缺陷
- 不用 AI 总结腔
